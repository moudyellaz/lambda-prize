# LP-0007: Monero–LEZ Atomic Swap

**`Logos Circle: N/A`**

## Overview

This prize funds a trustless Monero–LEZ atomic swap: a protocol where XMR and LEZ tokens exchange hands across chains without custodians, bridges, or wrapped tokens. Monero has no scripting system, so neither HTLCs nor standard adaptor signatures apply directly. The established solution — proven on Monero-Bitcoin mainnet by the COMIT network — combines Ed25519 adaptor signatures with cross-curve Discrete Log Equality (DLEQ) proofs to atomically transfer a Monero spend key share in exchange for funds on the other chain.

This prize extends that approach to LEZ, with the LEZ side implemented as a Risc0 guest program capable of verifying cross-curve DLEQ proofs natively in Rust.

## Motivation

Monero is the most widely used privacy-preserving cryptocurrency. A trustless XMR↔LEZ swap would give Logos users a direct, non-custodial on-ramp from the largest existing privacy coin ecosystem — without requiring Monero holders to expose themselves to a centralised exchange or a trusted bridge.

The cryptographic foundation is well-established: the h4sh3d/COMIT protocol ([paper](https://eprint.iacr.org/2020/1126.pdf)) has been live on Bitcoin-Monero mainnet since 2021, and a Monero-Starknet proof-of-concept demonstrates the same approach adapted to a non-Bitcoin chain. LEZ's use of Risc0 is a natural fit: cross-curve DLEQ verification can be written in ordinary Rust using `curve25519-dalek` and `secp256kFUN!`, run inside the zkVM guest without any custom circuit work.

A competitive prize is the right mechanism because the protocol is complex, the LEZ-side integration is novel, and implementation choices around key management, timelock handling, and the coordination protocol admit multiple valid approaches.

## Protocol Sketch

Monero has no scripting — funds cannot be locked to a hash or script condition. Instead, the protocol works by splitting the Monero spend key between the two parties and atomically selling one party's key share to the other.

The swap proceeds as follows (Alice holds XMR, Bob holds LEZ):

1. **Setup**: Alice and Bob each generate an Ed25519 key share (`a`, `b`). The XMR is sent to the address derived from the combined key `a + b` — neither party can spend unilaterally.
2. **Cross-curve binding**: Alice generates a secp256k1 adaptor point `T = a·G_secp` and provides a cross-curve DLEQ proof that `a` is the same scalar as in her Ed25519 share `A = a·G_ed`. Bob verifies this proof.
3. **Bob locks LEZ**: Bob locks LEZ tokens in a LEZ escrow program, redeemable by Alice upon revealing `a` (verified via the DLEQ proof on LEZ), or refundable to Bob after a timelock (enforced client-side — see note below).
4. **Alice claims LEZ**: Alice reveals `a` to the LEZ escrow and receives LEZ tokens. `a` is now observable on LEZ.
5. **Bob claims XMR**: Bob extracts `a` from Alice's LEZ claim and combines it with his own share `b` to reconstruct the full Monero spend key, then sweeps the XMR.

## Success Criteria

- [ ] A trustless XMR↔LEZ swap can be completed with no on-chain script on the Monero side — only standard Monero transactions.
- [ ] The LEZ escrow program correctly verifies the cross-curve DLEQ proof (secp256k1 ↔ Ed25519) and releases funds upon a valid reveal of the adaptor secret.
- [ ] The LEZ-side timelock refund is implemented. Note: LEZ does not yet have on-chain block time, so the LEZ-side timelock is enforced client-side (the same approach used in [eth-lez-atomic-swaps](https://github.com/logos-blockchain/eth-lez-atomic-swaps)).
- [ ] The two legs of the swap are atomic: either both complete or both refund. There is no state in which one party receives funds and the other does not.
- [ ] A reference integration is delivered: a working demo of a complete XMR↔LEZ swap on Monero stagenet and LEZ testnet.
- [ ] At least 5 complete XMR↔LEZ swaps are executed on Monero stagenet and LEZ testnet, involving at least 3 distinct counterparty pairs outside the submitting team.
- [ ] Full documentation and a clean public repository are delivered.

## Scope

### In Scope

- A Monero-side implementation: key share generation, XMR locking to a combined address, spend key reconstruction, and wallet sweep.
- A LEZ escrow program (Rust, Risc0) implementing cross-curve DLEQ proof verification (secp256k1 ↔ Ed25519) and conditional fund release.
- A cross-chain coordination CLI or script guiding both parties through the protocol.
- A reference integration on Monero stagenet and LEZ testnet demonstrating a complete swap.
- Documentation covering: protocol design, DLEQ proof construction, LEZ escrow design, cross-chain atomicity argument, timelock handling, security assumptions, and known limitations.

### Out of Scope

- Bitcoin-side integration — see LP-0006 for the BTC↔LEZ swap.
- Atomic swaps involving Monero subaddresses or multisig (standard address only).
- A polished consumer UI — a working demo is sufficient.
- Ongoing maintenance or security audits beyond initial delivery.
